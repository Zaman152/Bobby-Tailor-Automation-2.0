```python
async def _discover_and_add_linked_sheets(
    browser: StackCTBrowser,
    project_id: int,
    project_name: str,
    folder_id: Optional[int],
    all_extracted: List[dict],
    manifest: RunManifest,
    mpath: Path,
    screenshots_dir: Path,
    cached_screenshots: dict[int, Path],
    log: Callable,
    progress_callback: Optional[Callable],
    cancel_check: Optional[Callable[[], bool]],
) -> tuple[List[dict], List[dict], List[dict]]:
    """
    Pass 2a–2c: discover linked pages, capture them, analyze them.

    Returns (new_extracted, new_estimates, linked_meta) where:
      new_extracted: extraction dicts for linked pages
      new_estimates: calculator results for linked pages
      linked_meta: list of {page_id, sheet_name, ref_from} for report metadata
    """
```

**Implementation:**

Import at top of file (add to existing imports):
```python
from config import AUTO_INCLUDE_LINKED_SHEETS, MAX_LINKED_SHEETS
from linked_sheets import collect_unresolved_refs, match_ref_to_page
import stackct_store
```

Pass 2a — DISCOVER LINKS (no browser):
1. Call `collect_unresolved_refs(all_extracted, already_in_run=set(e.page_id for e in manifest.pages))`
2. Load catalog: `catalog = stackct_store.get_plans(project_id, folder_id)` — use folder_id if set, else None.
   If catalog is empty (e.g., DB not synced), log a WARNING and return ([], [], []).
3. For each ref record, call `match_ref_to_page(ref["ref_sheet"], catalog)`.
4. Build `linked_queue: list[dict]` — entries with keys `{page_id, sheet_name, ref_from}`.
   - Exclude page_ids already in manifest (capture_status any value).
   - Deduplicate by page_id (keep first).
   - Apply MAX_LINKED_SHEETS cap: truncate queue, log how many dropped.
5. If `not AUTO_INCLUDE_LINKED_SHEETS`:
   - Log "AUTO_INCLUDE_LINKED_SHEETS=false — skipping linked capture"
   - Return ([], [], [{"page_id": x["page_id"], "sheet_name": x["sheet_name"],
                        "ref_from": x["ref_from"], "suggested_only": True}
                       for x in linked_queue])
6. If linked_queue is empty: log "No new linked sheets to capture" and return ([], [], []).

Pass 2b — CAPTURE LINKED (browser required):
- Re-open browser only if linked_queue is non-empty: `await browser.start()` + `await browser.login()`.
  If login fails: log error and return ([], [], [{"error": "login_failed"}]).
- For each entry in linked_queue:
  - Check cancel_check(); break if cancelled.
  - Build screenshot_path: `screenshots_dir / f"linked_{entry['page_id']}_{_safe_sheet_filename(entry['sheet_name'])}.jpg"`
  - Try reuse from cached_screenshots first (same pattern as Pass 1).
  - Otherwise call `_capture_sheet_screenshot(browser, project_id, entry["page_id"], ...)`.
  - Append a `PageEntry` to `manifest.pages` with `source="linked_ref"` ... but `PageEntry` doesn't
    have `source` field yet — add it as Optional[str] with default None in capture_manifest.py
    (see Task 2 below, which runs in wave 3). For now, just use existing PageEntry fields;
    set `capture_status` to "ok" or "failed".
  - manifest.save(mpath) after each page.
- Close browser after capture pass.

Pass 2c — ANALYZE LINKED (no browser):
- For each newly appended manifest entry with capture_status=="ok":
  - Check cancel_check(); break if cancelled.
  - Call `analyze_drawing(str(screenshot_path), entry.sheet_name)`.
  - Set extracted["_page_id"] and extracted["_source_sheet"].
  - Call `apply_estimation_tables(extracted)`.
  - Append to new_extracted, new_estimates.
  - entry.analysis_status = "ok" or "failed".
  - manifest.save(mpath).
  - Log progress via log_callback.

Return (new_extracted, new_estimates, linked_meta) where linked_meta contains
`{page_id, sheet_name, ref_from}` for each successfully analyzed linked page.
