<task type="auto">
  <name>Task 2: Create settings.js for form logic</name>
  <files>static/settings.js (new)</files>
  <action>
Create `static/settings.js` with:

1. **`loadSettings()`:**
   - `fetch("/api/settings")` on page load
   - Populate form fields with returned values
   - For sensitive fields where `{KEY}_set` is true:
     - Show masked placeholder in input
     - Set `data-has-value="true"` attribute
   - For model dropdowns, select current value

2. **`handleSensitiveFieldFocus(input)`:**
   - When user focuses on a sensitive field that has existing value:
     - Clear the masked placeholder
     - Show empty input for new value entry
   - When user blurs without entering new value:
     - Restore masked placeholder if `data-has-value="true"`

3. **`saveSettings()`:**
   - Collect all form values
   - For sensitive fields:
     - If value is masked placeholder or empty, send empty string (preserve existing)
     - If user entered new value, send that value
   - `fetch("/api/settings", { method: "PUT", body: JSON.stringify(data) })`
   - On success:
     - Show success message
     - If `restart_required`, show warning: "Restart server to apply credential changes"
     - Reload settings to show updated state
   - On error:
     - Display error message from API
     - Highlight invalid fields if possible

4. **`initSettings()`:**
   - Call on DOMContentLoaded
   - `loadSettings()`
   - Attach event listeners for form submit, sensitive field focus/blur
  </action>

1. Page loads settings from API and displays them
2. Password fields show ••••••••, API key shows partial
3. Clicking password field clears it for new input
4. Submitting form calls PUT /api/settings
5. Success/error messages display correctly
  </verify>

  <name>Task 4: Implement sensitive field UX pattern</name>
  <files>static/settings.js, templates/settings.html</files>
  <action>
Implement the "click to edit" pattern for sensitive fields:

1. **Initial state:** Field shows masked value (e.g., "••••••••")
2. **On focus:** Clear field, show placeholder "Enter new password to change"
3. **On blur (empty):** Restore masked indicator if original value exists
4. **On blur (has input):** Keep user's input, will be submitted as new value
5. **On submit:** Empty field = preserve existing, non-empty = update

This prevents accidental overwrites while allowing intentional changes.

Add visual indicator (small lock icon or "Currently set" badge) for fields that have existing values.
