  <name>Task 1: Generalized COUNT_PROMPT + SCHEDULE_PROMPT</name>
  <files>claude_analyzer.py</files>
  <action>
**COUNT_PROMPT** — generic discrete-item counting (RESEARCH §11.7):
- Count ANY physical unit: bollards, columns, stairs, doors, windows, fixtures, panels, catch basins, manholes, equipment tags, ladders, lifts, hydrants, trees, light poles, etc.
- Universal rules:
  1. Numbers on dimension lines (with arrows/extension lines) = DIMENSION, never EA count
  2. "TYP", "O.C.", "MAX", "MIN" spacing annotations = spacing, not count
  3. Grid line labels (1,2,3 / A,B,C) = reference, not count
  4. Count visible symbols; use grid math when regular (rows × cols) with method="grid"
  5. quantity=null + confidence=low when uncertain — never fabricate
- Returns: `components[]`, `has_schedules` bool, `sheet_type` guess

**SCHEDULE_PROMPT** — any takeoff schedule:
- Door, window, finish, equipment, panel, pipe sizing schedules
- Extract exact QTY column values; classify rows by material/type columns (HM, WD, AL, PVC, etc.)
- table_purpose must be takeoff_schedule; use_for_takeoff=true only when qty column exists
- Do NOT invent row numbers; read only visible rows

**Do NOT include** project-specific examples (no "28 bollards", no "132 columns", no "886 LF").
Use generic examples: "count bollard symbols at loading dock", "read QTY column from door schedule".
