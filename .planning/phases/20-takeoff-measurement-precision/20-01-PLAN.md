Fix `_sheet_name_from_doc()` in `pdf_analyzer.py` to search only the title-block region (bottom-right 20%) of each PDF page instead of the full page text. This eliminates false-positive sheet IDs from ASTM standard references (E283, A156) that currently poison model routing and aggregation.

Purpose: Correct sheet identification is a prerequisite for every downstream fix — model routing, aggregation, and golden test matching all depend on accurate sheet names.

Output: Updated `_sheet_name_from_doc()` using PyMuPDF word bounding boxes; unit tests covering known false-positive cases.
</objective>

<task type="auto">
  <name>Task 1: Rewrite _sheet_name_from_doc with title-block region search</name>
  <files>pdf_analyzer.py</files>
  <action>
Replace the existing `_sheet_name_from_doc` function (lines 30–36) with a version that:

1. Uses `page.get_text("words")` to get per-word bounding boxes `(x0, y0, x1, y1, word, ...)`
2. Filters to words in the bottom-right region: `y > height * 0.80` AND `x > width * 0.55`
3. Joins those words and applies sheet-name regexes in priority order:
   - `r'\b([A-Z]\d+\.\d+)\b'` (e.g. A4.0, S1.2)
   - `r'\b([A-Z]-\d+)\b'` (e.g. C-4, A-101)
   - `r'\b([A-Z]\d{3})\b'` (e.g. A101)
4. Falls back to full-page search but filters out known noise patterns:
   - NOISE regex: `r'^(E283|A156|ASTM|UL\d|IBC|NFPA|ADA)'`
   - Uses `re.finditer` to skip noise matches and return first clean match
5. Final fallback returns `f"Page_{page_num + 1}"`

Use the exact implementation from RESEARCH.md §1. Do NOT change `_page_to_image`, `_sheet_name`, or `get_pdf_metadata` — they call `_sheet_name_from_doc` and need no changes.
  </action>
  <verify>python3 -c "from pdf_analyzer import _sheet_name_from_doc; print('import OK')"</verify>
  <done>_sheet_name_from_doc searches title-block region first; ASTM numbers filtered from fallback</done>
