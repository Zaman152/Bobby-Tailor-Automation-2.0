  <name>Task 2: GoldenValidator class + test_golden_takeoff.py</name>
  <files>tests/golden_validator.py, tests/test_golden_takeoff.py</files>
  <action>
Create `tests/golden_validator.py` with the `GoldenValidator` class from RESEARCH.md §9:

1. `__init__(golden_csv_path)` — loads golden CSV via `csv.DictReader`
2. `validate(takeoff_summary: list) -> dict` — returns `{pass, score, items, missing, extra}`
3. `_fuzzy_match(name, index)` — uses `difflib.get_close_matches(name, keys, n=1, cutoff=0.7)` after lowercasing + normalizing underscores/hyphens/spaces
4. Match modes:
   - `exact_or_within_1`: `ai_qty == g_qty` or `abs(ai_qty - g_qty) <= 1` (for EA counts)
   - `pct`: `abs(ai_qty - g_qty) / g_qty <= tolerance_pct / 100`
5. Score = passing_items / total_golden_items; `pass = score >= 0.97`
6. Return `missing` list (golden items not found in AI output) and per-item `error_pct`

Create `tests/test_golden_takeoff.py`:

1. Use `FIXTURES = Path(__file__).parent / "fixtures"` for fixture paths
2. `@pytest.mark.golden` marker on both tests
3. `@pytest.mark.skipif(not PDF_PATH.exists(), reason="Golden fixture PDF not present")` on each test
4. PDF paths: `tests/fixtures/crow_cass/crow_cass_plans.pdf` and `tests/fixtures/bobs_discount/bobs_discount_plans.pdf`
5. Each test calls `run_pdf_analysis()`, validates via `GoldenValidator`, asserts `score >= 0.97`
6. On failure, print per-item diagnostics: status, item name, golden qty, AI qty, error %

Also add a non-skipif unit test `test_golden_validator_logic` that verifies the validator math itself using synthetic data — ensures validator works even without PDFs.
