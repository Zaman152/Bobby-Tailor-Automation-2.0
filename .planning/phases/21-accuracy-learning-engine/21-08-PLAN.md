  - `semantic_labeling_schema(region_ids: list[str], wall_class_ids: list[str]) -> dict` — enums built from actual candidates; `additionalProperties: false`, all properties required (GA requirement)
  - `symbol_gallery_schema(symbol_ids: list[str], legend_labels: list[str]) -> dict` — maps SYM-n → legend label or "unknown"

  - `render_overlay(page, candidates: dict[str, tuple], kind_colors: dict = None, target_px: int = 1568) -> PIL.Image` — numbered markers/boxes per candidate, ≤40 markers (splits into multiple images beyond that, returning a list)
  - `build_crop_gallery(page, symbol_groups: list, legend_crop_rect=None, target_px: int = 1568) -> list[PIL.Image]` — contact sheet(s), ≤20 images/request budget respected
- **Module** `src/bobbytailor/vision/prompts.py` — existing pass prompts (EXTRACTION_PROMPT, COUNT_PROMPT, MEASURE_ADDENDUM, schedule/legend prompts) relocated verbatim from claude_analyzer + new `SEMANTIC_LABELING_PROMPT`, `SYMBOL_GALLERY_PROMPT`, `VERIFY_RETRY_PROMPT` (all instructing ID-only answers)
- **Changed function** `analyze_drawing(...)` in claude_analyzer.py — new optional params `output_schema: dict = None`, `images: list = None`; when schema given, uses `output_config={"format": {"type": "json_schema", "schema": ...}}` and parses the guaranteed-JSON response
- **New pass_type values**: `"semantic"`, `"symbol_gallery"`, `"verify_retry"` (routed via 21-05's `pick_model_for_pass`)

<threat_model>
ASVS L1 — API key handling + prompt-injection surface:
- **Prompt injection from PDF text (MEDIUM):** contained_texts/legend strings extracted from untrusted PDFs are embedded in prompts. Mitigation: schema-level enums mean injected instructions cannot change the response shape or add items outside the candidate set; prompts wrap PDF-derived strings in clearly delimited blocks.
- **API key (MEDIUM):** the client construction stays as-is (env-sourced); no key in logs — reuse the existing error-dict path that logs exception class, not request payloads.
- **Numeric leakage (design HIGH, mitigated):** any vector-sheet schema containing a number field defeats V3-ACC-01. Blocking acceptance: schema unit test asserts no integer/number-typed properties in vector-sheet schemas.
</threat_model>
