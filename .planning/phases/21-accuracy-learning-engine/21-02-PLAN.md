<threat_model>
ASVS L1 — this is the highest-risk plan for the "Flask endpoints" surface named in the phase security contribution:
- **Auth bypass via missed decorator (HIGH — blocking):** every moved route must retain `@login_required` exactly as in the flat app.py. Mitigation: Task 3 asserts route-by-route parity — an automated check compares the pre-split route→decorator inventory against the blueprint app's `app.url_map` and a login-less test client must get 302/401 on every protected endpoint.
- **CSRF regression (HIGH — blocking):** CSRFProtect must be initialized in create_app() and POST endpoints must keep rejecting missing X-CSRFToken. Test client POST without token must return 400.
- **Path traversal guards:** the `"/" in run_folder or ".." in run_folder` guards in reports/verify routes move verbatim — Task 2 acceptance asserts the guard string exists in the blueprint files.
- **Session config:** SESSION_COOKIE_SECURE conditional, SECRET_KEY sourcing, rate-limiter storage URI move unchanged into create_app().
</threat_model>
