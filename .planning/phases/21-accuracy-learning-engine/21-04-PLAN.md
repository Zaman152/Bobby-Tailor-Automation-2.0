<threat_model>
ASVS L1 — SQLite injection via learned strings is a named phase surface:
- **SQL injection through corrected_value/item_pattern (HIGH — blocking):** every value a human types flows into these tables and back out into prompts/aggregation. Mitigation: 100% parameterized queries (`?` placeholders) — Task 3 acceptance greps for f-string/`%`/`+` SQL construction and must find none; store module is the ONLY module allowed to touch `learning_*` tables (PATTERNS shared-pattern rule).
- **Stored-payload reflection (MEDIUM):** learned strings are later injected into prompts and UI. Store JSON values via `json.dumps` only; retrieval returns parsed values, never raw string interpolation into SQL. Length cap: `corrected_value`/`item_pattern` truncated at 2,000 chars on write (defense against payload stuffing).
- **DB path traversal (LOW):** `LEARNING_DB_PATH` comes from env, not user input; resolve and mkdir parents like STACKCT_DB_PATH.
- Decision: dedicated `learning.db` (not stackct.db) — isolates migration risk per RESEARCH §6 parenthetical; CONTEXT grants discretion.
</threat_model>
