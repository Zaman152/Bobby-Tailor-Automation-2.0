## Artifacts this phase produces

New symbols created by this plan:
- **Constant** `PROVENANCE_RANK: dict[str, int]` — `{"geometry": 5, "text_layer": 4, "schedule_table": 3, "learned": 2, "vision_ensemble": 1}`
- **Function** `fuse_sheet(deterministic_bundle: dict, vision_labels: dict, registry: dict[str, dict]) -> dict` — applies vision ID→label answers to deterministic candidates (wall classes → CMU/tilt-up names; SYM groups → legend items; rooms → labels; regions → main-plan pick); quantities always from the deterministic side; unknown/out-of-registry IDs dropped + flagged
- **Function** `merge_with_provenance(results: list[dict]) -> dict` — merge_passes successor: normalized-name dedup where the higher PROVENANCE_RANK wins; equal rank → higher confidence; every component gains `"provenance"` and keeps `"needs_review"`/vote metadata
- **Item field** `provenance: str` on every component dict (consumed by aggregator/reporter in 21-13)
- **Class** `ReplayAnalyzer` in tests/replay_analyzer.py — `__init__(fixture_dir, record: bool = False)`; keys recorded responses by SHA256 of (model, pass_type, prompt text, image count); record mode wraps the real analyzer once, replay mode raises KeyError on unknown prompts (test determinism); injectable via `TakeoffPipeline(analyzer=ReplayAnalyzer(...))`
- **Fixture dir** `tests/fixtures/replay/` — recorded structured-output JSON responses

<threat_model>
ASVS L1:
- **Grounding bypass (design HIGH, mitigated):** if a vision answer could introduce an item with its own quantity on a vector sheet, the whole accuracy contract fails. Mitigations: fuse_sheet only accepts registry IDs (out-of-set → dropped + `fusion_rejected` flag list); merge_with_provenance refuses to let a vision_ensemble row replace any higher-rank row's quantity (assert + test).
- **Replay fixture poisoning (LOW):** replay fixtures are repo-committed test data; record mode is developer-invoked only (env `REPLAY_RECORD=1`), never active in CI.
- No new endpoints/SQL.
</threat_model>
