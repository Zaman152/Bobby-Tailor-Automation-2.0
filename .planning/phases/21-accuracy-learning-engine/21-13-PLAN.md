New symbols created by this plan:
- **Dataclass** `RunSettings` in takeoff_pipeline.py — fields: `deterministic_enabled: bool = True`, `ensemble_n: int` (from config ENSEMBLE_N), `learning_enabled: bool = True`, `budget_usd: float` (from API_BUDGET_PER_RUN_USD), `verify_retry_enabled: bool = True`, `raster_force: bool = False`; classmethod `from_env()`
- **Function** `is_vector_sheet(page) -> bool` — the RESEARCH §2 heuristic: NOT (< 200 drawing items AND < 100 words)
- **Method/flow** `TakeoffPipeline.run_sheet` gains the deterministic-first branch: builds the deterministic bundle (samples → viewports → per-viewport scales → walls/rooms/symbols/tables), renders overlays/galleries, runs the semantic vision pass, calls `fuse_sheet`, then `merge_with_provenance`; raster sheets → existing extraction passes wrapped in `run_ensemble` with `measurement_mode="raster"` and all quantities flagged
- **Result fields** in sheet results/takeoff.json: `measurement_mode`, `scale_confidence` (per-viewport ViewportScale.to_dict list), `provenance` per item, `fusion_rejected`, `budget` block `{cap_usd, spent_usd, exhausted}`
- **Reporter additions**: summary.txt "SCALE CONFIDENCE" and "PROVENANCE BREAKDOWN" sections; takeoff.json carries the new fields
- **Constructor change**: `TakeoffPipeline(analyzer=None, settings: RunSettings = None)` — settings defaults to from_env()
