2. Create `stackct_store.py` using **stdlib `sqlite3`** only (no SQLAlchemy — keeps deps unchanged):
   - `get_connection()` with `row_factory=sqlite3.Row`, `PRAGMA foreign_keys=ON`
   - `init_schema(conn)` executing tables from 13-RESEARCH.md: `projects`, `project_plans`, `sync_runs`, `cache_metadata`

3. CRUD functions (typed dict returns, no ORM):
   - `upsert_projects(projects: list[dict], synced_at: str) -> int`
   - `upsert_plans(stackct_id: int, plans: list[dict], synced_at: str) -> int` — updates `projects.sheet_count` and `plans_synced_at`
   - `list_projects() -> list[dict]` — `{id, name, sheet_count, plans_synced_at, from_cache}` shape compatible with UI
   - `get_plans(stackct_id: int) -> list[dict]` — `{page_id, sheet_name}`
   - `get_sheet_counts() -> dict[int, int]` — `{project_id: count}` for non-null sheet_count only
   - `is_projects_fresh() -> bool` / `is_plans_fresh(stackct_id: int) -> bool` using TTL from config
   - `record_sync_run(...)` / `finish_sync_run(...)` for audit table

4. Use `threading.Lock` **inside store only for writes** OR document that callers use sync-layer lock — prefer caller lock in 13-02; store methods stay reentrant for reads.
